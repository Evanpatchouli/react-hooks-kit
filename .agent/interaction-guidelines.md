# 交互基本准则

## 沟通方式

1. **语言**: 使用中文进行交流
2. **风格**: 简洁、专业、直接
3. **表情符号**: 除非明确要求，否则不使用

## 工作原则

### 代码修改
- 在修改代码前，必须先阅读相关文件
- 优先编辑现有文件，而非创建新文件
- 避免过度工程化，只做必要的修改
- 不添加未被要求的功能或重构

### 文档管理
- 不主动创建文档文件（除非明确要求）
- 保持文档简洁实用

### 任务执行
- 遇到不确定的地方，主动询问
- 完成任务后简要说明做了什么
- 记录重要的决策和讨论

## 项目特定准则

### 版本管理
- 遵循项目的版本号规则（x.y.z）
- 提交代码时使用规范的 commit message

### 代码质量
- 保持 TypeScript 类型安全
- 注意安全漏洞（XSS、注入等）
- 编写简洁、可维护的代码

### Hook 注释标准
- 所有 hook 必须遵循 `.claude/hook-comment.md` 中定义的注释标准
- 注释必须包含：描述、参数、返回值、用法、示例、常见问题
- 参考 `useMeta` 的注释格式

## 记录习惯

每天工作结束时，在 `.claude/daily/` 目录下创建或更新以日期命名的文件（如 `2026-02-05.md`）：
- 记录完成的工作
- 记录遇到的问题和解决方案
- 更新待办事项
- 记录项目当前状态
